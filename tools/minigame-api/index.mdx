# Minigame API

The **Minigame API** (`library-minigame-api`) provides a standardized framework for building minigames on the Grounds platform. It abstracts away boilerplate like game state management, team handling, event systems, and statistics tracking — so you can focus on gameplay logic.

## Overview

The API is split into two modules:

| Module | Artifact | Description |
|--------|----------|-------------|
| **core** | `library-minigame-api-core` | Platform-independent interfaces, events, team management, stats collector |
| **paper** | `library-minigame-api-paper` | Paper/Bukkit implementation with scheduler-based timers and lifecycle integration |

## What it provides

- **Lifecycle hooks** — Override `onStartingPhase()`, `onRunningPhase()`, `onEndingPhase()` instead of wiring up manual state machines
- **Player tracking** — Thread-safe set of current players, automatically maintained by the base class
- **Team management** — Create teams, assign players, track eliminations
- **Event bus** — Publish and subscribe to minigame events with unsubscribe support
- **Statistics collector** — Thread-safe tracking of kills, deaths, and custom stats for batch submission
- **Timers** — Named, pausable timers with multiple tick and completion handlers

## Architecture

The Minigame API sits between your gameplay code and the Grounds infrastructure:

```
┌─────────────────────────────────┐
│  Your Minigame (e.g. BedWars)   │  ← You write this
├─────────────────────────────────┤
│  library-minigame-api (paper)   │  ← Lifecycle, Teams, Events
├─────────────────────────────────┤
│  plugin-match                   │  ← Matchmaking, Queue, Allocation
├─────────────────────────────────┤
│  plugin-agones                  │  ← Server Lifecycle, K8s
└─────────────────────────────────┘
```
