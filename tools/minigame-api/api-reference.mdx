# API Reference

## Core Module (`gg.grounds.minigame`)

### Minigame

Abstract base class for all minigames. Extend `PaperMinigame` for Paper implementations.

```kotlin
abstract class Minigame {
    abstract val id: String
    abstract val displayName: String
    open val variant: GameVariant             // Default: CASUAL

    val stateManager: GameStateManager        // Initialized during PaperMinigame.initialize()
    val teamManager: TeamManager              // Initialized during PaperMinigame.initialize()
    val eventBus: MinigameEventBus            // Initialized during PaperMinigame.initialize()
    val stats: MatchStatsCollector
    val players: Set<UUID>                    // Read-only view of current players

    // Standard lifecycle
    open fun onLoad()
    open fun onEnable()
    open fun onDisable()

    // Phase entry hooks
    open fun onWaitingPhase()
    open fun onStartingPhase()
    open fun onRunningPhase()
    open fun onEndingPhase()
    open fun onShutdownPhase()

    // Phase exit hooks
    open fun onExitWaiting()
    open fun onExitStarting()
    open fun onExitRunning()
    open fun onExitEnding()
}
```

<Warning>
  `stateManager`, `teamManager`, and `eventBus` are initialized during `PaperMinigame.initialize()` and must not be accessed before that call completes (e.g. not in constructors or property initializers).
</Warning>

### MinigameEventBus

```kotlin
interface MinigameEventBus {
    fun <T : MinigameEvent> subscribe(
        eventClass: KClass<T>,
        priority: EventPriority = EventPriority.NORMAL,
        handler: (T) -> Unit,

    fun publish(event: MinigameEvent)

// Extension function for inline subscription
```

## Paper Module (`gg.grounds.minigame.paper`)

### PaperMinigame

```kotlin
abstract class PaperMinigame(val plugin: JavaPlugin) : Minigame() {
    fun initialize()                    // Set up managers, call onLoad + onEnable
    fun shutdown()                      // Transition to SHUTDOWN, call onDisable

    // Tracks player in `players` set, publishes event. Override in subclasses for custom logic.
    open fun handlePlayerJoin(playerId: UUID)
    open fun handlePlayerLeave(playerId: UUID)
    open fun handlePlayerDeath(playerId: UUID, killerId: UUID? = null, cause: String? = null)
}
```

### PaperGameStateManager

Paper implementation of `GameStateManager`. Automatically calls lifecycle hooks on the `Minigame` when transitioning between phases, and publishes corresponding events to the event bus.

### PaperGameTimer

Paper implementation of `GameTimer` using `BukkitScheduler.scheduleSyncRepeatingTask`. Ticks every second (20 ticks).

## Default Implementations

The `core` module includes platform-independent default implementations:

| Class | Implements | Notes |
|-------|-----------|-------|
| `SimpleEventBus` | `MinigameEventBus` | Synchronous, priority-ordered dispatch. Sorted on subscribe for efficient publish. |
| `SimpleTeam` | `Team` | Thread-safe via `ConcurrentHashMap` |
| `SimpleTeamManager` | `TeamManager` | Thread-safe, tracks eliminations separately |
