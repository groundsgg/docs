# Root Plugin (gg.grounds.root)

The `gg.grounds.root` plugin provides shared repository configuration for Grounds projects. It sets up common build conventions, code quality tools, and publishing configurations that are used across all Grounds projects.

## Features

The root plugin automatically configures:

- **Kotlin Language Support**: Kotlin language plugin and standard library dependencies
- **Publishing Setup**: GitHub Packages publishing configuration
- **Code Formatting**: Spotless integration for consistent code formatting
- **Test Logging**: Standardized test output and logging configuration
- **Common Build Conventions**: Shared build settings and conventions

## When to Use

Use the root plugin when:

- Setting up a new Grounds project
- Managing a multi-module project where you want shared configuration
- You need consistent build conventions across multiple projects
- You want to reduce boilerplate configuration code

## Usage

Apply the plugin in your root `build.gradle.kts`:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.root") version "0.1.1"
}
```

### Multi-Module Projects

For multi-module projects, apply the root plugin at the root level:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.root") version "0.1.1"
}

allprojects {
    group = "gg.grounds"
    version = "1.0.0"
}
```

## What It Configures

### Kotlin Support

The plugin automatically applies and configures:

- Kotlin JVM plugin
- Kotlin standard library dependencies
- Kotlin compiler options

### Publishing

Configures publishing to GitHub Packages:

- Maven publication setup
- Repository configuration
- Artifact metadata

### Code Formatting

Integrates Spotless for code formatting:

- Kotlin formatting rules
- License header management
- Automatic formatting on build

### Test Configuration

Standardizes test execution:

- Test logging output
- Test report generation
- Test task configuration

## Example Project Structure

A typical project using the root plugin:

```kotlin settings.gradle.kts
pluginManagement {
    repositories {
        maven {
            url = uri("https://maven.pkg.github.com/groundsgg/*")
            credentials {
                username = providers.gradleProperty("github.user").get()
                password = providers.gradleProperty("github.token").get()
            }
        }
        gradlePluginPortal()
    }
}

rootProject.name = "my-grounds-project"
```

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.root") version "0.1.1"
}

group = "gg.grounds"
version = "1.0.0"
```

## Benefits

- **Consistency**: All projects follow the same build conventions
- **Less Configuration**: No need to manually set up Kotlin, publishing, or formatting
- **Maintainability**: Updates to conventions are centralized in the plugin
- **Developer Experience**: New projects are ready to use immediately
