# Configuration

Both `gg.grounds.paper` and `gg.grounds.velocity` plugins support the `grounds` extension block for additional configuration. This allows you to customize plugin behavior to match your project's needs.

## Grounds Extension

The `grounds` extension is available in both Paper and Velocity plugins. Configure it in your `build.gradle.kts`:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

grounds {
    // Configuration options here
}
```

## Configuration Properties

### addDependencies

<ParamField body="addDependencies" type="boolean" default="true">
Controls whether the plugin automatically adds Paper or Velocity API dependencies to your project.

- **Default**: `true`
- **When `true`**: The plugin automatically adds the appropriate API dependency (Paper API for Paper plugin, Velocity API for Velocity plugin)
- **When `false`**: You must manually manage API dependencies in your `dependencies` block

Use this option when you need to:
- Use a specific version of the API
- Manage dependencies manually
- Use a different API source
</ParamField>

## Configuration Examples

### Default Configuration (Automatic Dependencies)

By default, the plugin automatically adds API dependencies:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

// addDependencies defaults to true, so Paper API is automatically added
```

This is equivalent to:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

grounds {
    addDependencies.set(true)
}
```

### Manual Dependency Management

To manage dependencies manually, disable automatic dependency addition:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

grounds {
    addDependencies.set(false)
}

dependencies {
    compileOnly("io.papermc.paper:paper-api:1.20.4-R0.1-SNAPSHOT")
    // Add other dependencies as needed
}
```

### Paper Plugin Configuration

Example Paper plugin with custom configuration:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

group = "gg.grounds"
version = "1.0.0"

grounds {
    addDependencies.set(true)
}

dependencies {
    // Additional dependencies
    implementation("com.google.guava:guava:32.1.3-jre")
}
```

### Velocity Plugin Configuration

Example Velocity plugin with custom configuration:

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.velocity") version "0.1.1"
}

group = "gg.grounds"
version = "1.0.0"

grounds {
    addDependencies.set(true)
}

dependencies {
    // Additional dependencies
    implementation("com.google.guava:guava:32.1.3-jre")
}
```

## Best Practices

### When to Use Automatic Dependencies

Use `addDependencies.set(true)` (default) when:

- You want the plugin to manage API versions automatically
- You're starting a new project
- You want to follow plugin best practices
- You don't need a specific API version

### When to Use Manual Dependency Management

Use `addDependencies.set(false)` when:

- You need a specific API version
- You're migrating from a manual setup
- You need to use a different API source
- You have complex dependency requirements

### Version Management

When managing dependencies manually, consider:

- Using version catalogs for version management
- Keeping API versions up to date
- Testing with different API versions before upgrading

Example with version catalog:

```kotlin gradle/libs.versions.toml
[versions]
paper = "1.20.4-R0.1-SNAPSHOT"

[libraries]
paper-api = { module = "io.papermc.paper:paper-api", version.ref = "paper" }
```

```kotlin build.gradle.kts
plugins {
    id("gg.grounds.paper") version "0.1.1"
}

grounds {
    addDependencies.set(false)
}

dependencies {
    compileOnly(libs.paper.api)
}
```

## Root Plugin

The `gg.grounds.root` plugin does not support the `grounds` extension. It provides fixed, standardized configuration that applies to all projects using it.

## Troubleshooting

### Dependencies Not Found

If you're managing dependencies manually and encounter dependency resolution errors:

1. Verify the repository is correctly configured
2. Check that the API version exists
3. Ensure your dependency coordinates are correct

### Version Conflicts

If you encounter version conflicts:

1. Use `addDependencies.set(false)` to manage versions manually
2. Check for conflicting dependencies in your dependency tree
3. Use `./gradlew dependencies` to inspect the dependency graph
